# First Screen UI Polish Plan v1

작성 시각: 2026-06-14 19:57:09

## 1. 목적

현재 연결된 1차 시각 요소를 기준으로 첫 화면 UI를 정리하는 계획이다.

이번 Plan은 구현 계획만 작성한다.
코드 수정, data 수정, public/assets 수정, 이미지 생성, git 작업은 하지 않는다.

## 2. 현재 연결된 시각 요소

현재 게임 첫 화면에는 아래 요소가 연결되어 있다.

```text
오계장 플레이어 스프라이트
Batch 1 몬스터 이미지
Batch 2 장착 장비 HUD 아이콘
Batch 3 기본 스킬/타격 이펙트
Batch 5 지역 배경
우측 메뉴 접기/펼침 UI
전투 제어 UI: 수동 / 오토 / x1.5 / x2
```

이제 단순 기능 표시가 아니라, 유저가 첫 화면을 봤을 때 `자동 전투 중인 모바일 방치형 RPG`라는 느낌을 바로 받을 수 있게 정리해야 한다.

## 3. 현재 첫 화면 구성 요약

현재 첫 화면은 크게 아래 영역으로 나뉜다.

```text
상단 왼쪽: 플레이어 이름 / 레벨 / HP
상단 중앙: 스테이지 / 지역 / 목표
상단 오른쪽: 골드 / 다이아 / 상점 / 편지함
중앙: 오계장과 몬스터 전투 장면
왼쪽 정보 패널: 공격 / 방어 / EXP / 장비 보너스 / 인벤토리 요약
오른쪽: 접기/펼침 가능한 메뉴
하단 왼쪽: 전투 제어 UI
하단 중앙: 스킬 슬롯 6개
```

현재 가장 큰 문제는 `기능은 많아졌지만 첫 화면의 시선 흐름이 아직 정리되지 않았다`는 점이다.

## 4. 첫 화면 우선순위

첫 화면에서 가장 중요한 요소 순서:

1. 중앙 전투 장면
2. 현재 플레이어 HP와 몬스터 HP
3. 현재 스테이지 / 지역 / 목표
4. 스킬 슬롯과 전투 제어 상태
5. 성장 버튼: 스킬 / 장비 / 가방 / 퀘스트
6. 골드 / 다이아 / 상점 / 편지함
7. 상세 스탯 / 장비 보너스 / 인벤토리 요약

1차 Polish에서는 `상세 정보`보다 `전투 장면과 핵심 조작`을 우선한다.

## 5. 중앙 전투 영역 정리 방향

목표:

- 오계장과 몬스터가 가장 먼저 보여야 한다.
- 지역 배경은 분위기를 만들되 캐릭터를 묻지 않아야 한다.
- HP bar는 캐릭터/몬스터 위에 가깝게 배치한다.
- 스킬 이펙트가 너무 크게 화면을 덮지 않도록 scale을 조정한다.

정리 방향:

- 중앙 전투 영역의 반투명 박스가 배경과 캐릭터를 과하게 덮으면 축소 또는 제거한다.
- 플레이어와 몬스터 위치를 더 명확히 좌우로 나눈다.
- HP bar와 이름 라벨은 캐릭터 바로 위에 붙인다.
- 몬스터 이미지 크기는 role 기준으로 유지하되, 보스만 더 크게 둔다.
- 스킬 이펙트는 몬스터 쪽 중심에 뜨되 하단 스킬 슬롯을 가리지 않게 한다.

## 6. 상단 정보 영역 정리 방향

상단은 `짧고 명확한 현재 상태`만 보여준다.

유지:

- 플레이어 이름: 오계장
- 레벨
- HP
- 현재 스테이지
- 지역
- 목표
- 골드
- 다이아
- 상점
- 편지함

정리 방향:

- 스테이지 표기는 `스테이지 4-1`처럼 짧게 유지한다.
- 지역명과 목표는 한 줄 또는 두 줄 이내로 제한한다.
- 골드/다이아 패널은 너무 크지 않게 유지한다.
- 상점/편지함은 우측 상단 보조 버튼으로 유지하되, 전투 영역을 침범하지 않게 한다.

## 7. 우측 메뉴 영역 정리 방향

우측 메뉴는 성장/관리 기능 진입점이다.

대상:

```text
스킬
장비
가방
퀘스트
```

정리 방향:

- 접힘 상태: 메뉴 버튼 1개만 표시한다.
- 펼침 상태: 스킬 / 장비 / 가방 / 퀘스트 아이콘을 세로 정렬한다.
- 버튼 배경은 통짜 패널로 묶는다.
- 이상한 삼각형, 구분용 실선, 불필요한 장식은 제거한다.
- 아이콘과 라벨 중심선을 맞춘다.
- 현재 선택된 항목만 하이라이트한다.
- 1차에서는 실제 패널 열기 기능은 연결하지 않는다.

중요:

- 우측 메뉴는 전투 로그 영역을 대체한다.
- 긴 Battle Log는 첫 화면에서 제외한다.

## 8. 하단 스킬 슬롯 / 전투 제어 정리 방향

하단은 전투 조작과 스킬 상태를 보여준다.

전투 제어:

```text
수동
오토
x1.5
x2
```

정리 방향:

- 접힘 상태: 현재 선택 모드만 표시한다.
- 펼침 상태: 수동 / 오토 / x1.5 / x2를 보여준다.
- x1.5 / x2는 잠금 후보로 보일 수 있게 한다.
- 실제 전투 속도 반영은 아직 하지 않는다.

스킬 슬롯:

- 6칸 유지
- 연결된 스킬은 앞쪽 슬롯에 표시
- 잠긴 슬롯은 명확하게 잠금 표시
- 쿨타임 숫자는 작지만 읽히게 유지
- 이펙트가 슬롯 위를 과하게 덮지 않게 한다.

## 9. 장비 아이콘 표시 위치 정리 방향

현재 장착 장비 아이콘은 왼쪽 정보 패널 안에 표시된다.

정리 방향:

- 6슬롯 구조 유지
- 아이콘 크기는 작지만 슬롯 간격을 정리한다.
- 장비 보너스 텍스트는 너무 길면 축약한다.
- 첫 화면에서는 `무슨 장비를 꼈는지`보다 `장비가 적용 중인지`를 보여주는 정도로 제한한다.

후속 작업 후보:

- 장비 버튼을 눌렀을 때 전용 장비 패널에서 상세 장비 정보를 보여준다.

## 10. 삭제하거나 축소해야 할 정보

첫 화면에서 축소 또는 숨김 후보:

- 긴 Battle Log
- 전체 인벤토리 목록
- 상세 보유 장비 목록
- 지나치게 긴 장비 보너스 설명
- 전투 로그성 문자열
- 내부 id 기반 문자열
- 너무 많은 영어 UI 텍스트

유지하되 짧게 보여줄 정보:

- 공격 / 방어
- EXP
- 장비 보너스 요약
- 보유 아이템 수 / 장비 수

## 11. 아직 연결하지 않을 기능

이번 Polish 단계에서 연결하지 않을 기능:

- 실제 스킬 패널 열기
- 실제 장비 패널 열기
- 실제 가방 패널 열기
- 실제 퀘스트 패널 열기
- 실제 수동 전투 모드
- 실제 x1.5 / x2 배속
- 보상 팝업
- 스테이지 클리어 팝업
- 레벨업 연출
- 배경 전환 애니메이션

## 12. 수정 후보 파일

구현 승인 시 수정 후보:

```text
src/ui/Hud.ts
src/scenes/GameScene.ts
docs/UI_SYSTEM.md
docs/PROJECT_STATUS.md
files/Walkthrough/YYYY-MM-DD_HHMMSS_FirstScreenUIPolish_Walkthrough_v1.md
```

가능하면 `Hud.ts` 중심으로 수정한다.
`GameScene.ts`는 표시용 좌표나 effect scale 조정이 꼭 필요한 경우에만 최소 수정한다.

## 13. 수정 금지 파일

기본 수정 금지:

```text
src/systems/*
data/*.json
public/assets/**
README.md
docs/AGENT_HANDOFF.md
기존 보류 plan / Walkthrough 파일
```

## 14. 검증 방법

구현 후 실행:

```powershell
npm.cmd run typecheck
npm.cmd run build
```

금지 영역 확인:

```powershell
git diff -- src/systems data public/assets
git status --short
```

## 15. Visual Validation Deferred 기준

브라우저 확인을 바로 하지 않을 경우 아래 항목은 deferred로 기록한다.

- 첫 화면에서 오계장과 몬스터가 먼저 보이는지
- 지역 배경이 캐릭터를 묻지 않는지
- 우측 메뉴 접힘/펼침이 시각적으로 이해되는지
- 전투 제어 접힘/펼침이 시각적으로 이해되는지
- 스킬 슬롯 6개가 유지되는지
- HP bar, stage, gold, level이 겹치지 않는지
- 이펙트가 화면을 과하게 덮지 않는지

## 16. 구현 전 승인 체크포인트

구현 전 사용자가 승인해야 할 항목:

- 긴 Battle Log를 첫 화면에서 계속 제외할지
- 왼쪽 정보 패널을 더 줄일지
- 우측 메뉴를 기본 펼침으로 둘지, 기본 접힘으로 둘지
- 전투 제어를 기본 접힘으로 둘지, 기본 펼침으로 둘지
- 스킬 이펙트 scale을 줄일지
- 배경 위 중앙 반투명 전투 박스를 제거할지 유지할지

## 17. 추천 1차 구현 범위

추천 1차 구현:

```text
Hud.ts 중심 UI polish
-> 중앙 반투명 박스 축소 또는 제거
-> 우측 메뉴 패널 정렬 보정
-> 전투 제어 패널 정렬 보정
-> 스킬 슬롯 6칸 유지 확인
-> 장비 슬롯 간격 정리
-> 긴 텍스트 축약
-> Visual Validation deferred
```

실제 전투 로직, 보상 로직, 데이터 구조는 건드리지 않는다.
