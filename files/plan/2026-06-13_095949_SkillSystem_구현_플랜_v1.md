## 2. SkillSystem (스킬 시스템)이 담당할 책임

`SkillSystem (스킬 시스템)`은 아래 책임만 담당한다.

```text
Skill cooldown (스킬 쿨타임) 관리
Auto trigger condition (자동 발동 조건) 판단
Skill damage calculation (스킬 피해 계산)
SkillResult (스킬 결과) 반환
SkillState (스킬 상태) 저장 데이터 변환
```

담당하지 않는 것:

```text
경험치 지급
골드 지급
아이템 지급
인벤토리 수정
장비 스탯 계산
플레이어 기본 스탯 영구 변경
스테이지 진행
```

## 3. SkillData (스킬 정적 데이터) 구조

새 데이터 파일:

```text

```

권장 타입:

```ts
interface SkillData {
  id: string;
  name: string;
  description: string;
  trigger: "auto";
  target: "currentMonster";
  cooldownMs: number;
  damageMultiplier: number;
  flatDamage: number;
  requiredLevel: number;
}
```

## 4. SkillState (스킬 저장/런타임 상태) 구조

저장 데이터:

```ts
interface SkillState {
  unlockedSkillIds: string[];
  equippedSkillIds: string[];
}
```

런타임 전용 데이터:

```ts
interface SkillRuntimeState {
  cooldownRemainingBySkillId: Record<string, number>;
}
```

MVP 1 원칙:

```text
SkillState (스킬 저장 상태)는 저장한다.
SkillRuntimeState (스킬 런타임 상태)는 저장하지 않는다.
저장/불러오기 후 스킬 쿨타임은 0부터 시작한다.
```

## 5. 스킬 슬롯 또는 기본 스킬 보유 방식

MVP 1은 단순한 2-slot structure (2슬롯 구조)를 사용한다.

```text
slot 1: trainee_slash
slot 2: heavy_training_strike
```

기본 fallback:

```ts
skills: saved?.skills ?? {
  unlockedSkillIds: ["trainee_slash", "heavy_training_strike"],
  equippedSkillIds: ["trainee_slash", "heavy_training_strike"]
}
```

중요:

```text
requiredLevel (요구 레벨)을 만족하지 않는 스킬은 장착되어 있어도 자동 발동하지 않는다.
```

## 6. 자동 발동 조건

자동 발동 조건:

```text
monster.isDead === false
player.hp > 0
skill is equipped
skill is unlocked
player.level >= skill.requiredLevel
cooldownRemaining <= 0
```

MVP 1 정책:

```text
한 update tick에서 발동 가능한 스킬이 여러 개여도 하나만 발동한다.
equippedSkillIds 순서대로 검사한다.
```

## 7. Cooldown (쿨타임) 처리 방식

매 update마다 쿨타임을 감소시킨다.

```ts
cooldownRemaining = Math.max(0, cooldownRemaining - deltaMs);
```

스킬 발동 성공 시:

```ts
cooldownRemainingBySkillId[skill.id] = skill.cooldownMs;
```

쿨타임 표시용 view:

```ts
interface SkillCooldownView {
  skillId: string;
  name: string;
  ready: boolean;
  cooldownRemainingMs: number;
}
```

## 8. 스킬 피해 계산 방식

스킬 피해는 `EffectivePlayerStats (최종 플레이어 스탯)`를 기준으로 계산한다.

```ts
skillDamage = Math.max(
  1,
  Math.floor(effectiveStats.attack * skill.damageMultiplier + skill.flatDamage - monster.data.defense)
);
```

중요 원칙:

```text
effectiveStats.attack = player.attack + equipment bonus
SkillSystem은 player.attack을 직접 수정하지 않는다.
SkillSystem은 EquipmentSystem의 계산 결과를 읽기만 한다.
```

## 9. CombatSystem (전투 시스템)과 연결 방식

권장 흐름:

```text
GameScene.update
-> effectiveStats = EquipmentSystem.calculateEffectiveStats(player)
-> skillResult = SkillSystem.update(delta, player, effectiveStats, monster)
-> if skillResult.defeated, handleMonsterDefeat()
-> else CombatSystem.update(delta, player, effectiveStats, monster)
-> if combatResult.defeated, handleMonsterDefeat()
```

MVP 1에서는 `CombatSystem (전투 시스템)` 내부에 스킬 로직을 넣지 않는다.

이유:

```text
CombatSystem은 기본 공격과 반격 계산을 유지한다.
SkillSystem은 자동 발동 스킬 계산을 별도 시스템으로 유지한다.
보상 흐름은 기존 handleMonsterDefeat -> RewardResolver -> RewardSystem을 그대로 재사용한다.
```

## 10. PlayerGrowthSystem / EquipmentSystem과 충돌하지 않는 구조

충돌 방지 규칙:

```text
PlayerGrowthSystem (플레이어 성장 시스템)
= 기본 스탯 영구 성장만 담당

EquipmentSystem (장비 시스템)
= 장비 보너스와 EffectivePlayerStats 계산만 담당

SkillSystem (스킬 시스템)
= EffectivePlayerStats를 읽어서 일시적인 피해량 계산만 담당
```

금지 코드:

```ts
player.attack += 10;
player.defense += 5;
player.maxHp += 20;
```

허용:

```ts
const damage = calculateSkillDamage(skill, effectiveStats, monster);
```

## 11. SaveSystem에 저장해야 할 데이터

`SaveData`에 추가:

```ts
skills?: SkillState;
```

저장 시:

```ts
skills: skillSystem.toState()
```

불러오기 fallback:

```ts
const skillState = saved?.skills ?? {
  unlockedSkillIds: ["trainee_slash", "heavy_training_strike"],
  equippedSkillIds: ["trainee_slash", "heavy_training_strike"]
};
```

저장하지 않는 것:

```text
cooldownRemainingBySkillId
skill runtime timers
temporary skill effects
```

## 16. 예상 리스크

중복 처치 처리:

```text
SkillSystem과 CombatSystem이 같은 update tick에서 동시에 몬스터 처치를 처리하면 보상이 중복 지급될 수 있다.
해결: 스킬로 처치되면 해당 tick에서는 CombatSystem.update를 호출하지 않는다.
```

보상 흐름 중복:

```text
스킬 처치도 반드시 기존 handleMonsterDefeat() 흐름을 재사용해야 한다.
RewardResolver / RewardSystem을 별도로 다시 호출하지 않는다.
```

HUD 공간 부족:

```text
MVP 1에서는 텍스트만 간단히 추가한다.
별도 스킬 패널은 추후 UI 리팩토링에서 분리한다.
```

쿨타임 저장 여부:

```text
쿨타임은 저장하지 않는다.
재접속 직후 스킬이 바로 준비되는 것은 MVP 1에서 허용한다.
```

과설계 위험:

```text
스킬 트리, 스킬 강화, 속성 시스템, 광역 스킬, 상태이상은 MVP 1에서 제외한다.
```
